# 英语阅读项目数据库数据填写规范

##  一、总则 

1. 字符编码：所有文本字段统一使用`UTF-8`，避免中文/特殊字符乱码； 
2. 字段类型：严格匹配定义，数值型仅填数字、枚举型仅填指定值，无多余符号；
3. 唯一性约束：`slug`（文章/标签）、`email`（用户）、联合主键（收藏/答题/阅读记录等）**严禁重复**； 
4. 时间格式：所有时间字段统一为`YYYY-MM-DD HH:MM:SS`，未指定时可填`CURRENT_TIMESTAMP`； 
5. 软删除：未删除数据的`deleted_at`字段**必须为NULL**，删除时填充对应时间； 
6. 外键关联：`article_id`/`user_id`/`tag_id`等关联字段，必须存在于对应主表中，避免外键冲突。先填写**父表（用户表、文章表）**

## 二、分表填写规范

1. users（用户表） ： 填写数量：**3-5条**
2. tags（标签表） ：多填充一些标签，10+，与对应文章**关联**
3. articles（文章表） ：
   1. 填写数量：**5个subject（具体哪个学科数据库内有写）各6-7篇**，总计30-35篇；
   2. 文章来源：严格审查文章，不能有语法拼写等错误，干脆使用**四六级真题、雅思真题**
   3. 难度分布：单subject内按**Easy:Intermediate:Advanced=2:2:2/3**均匀分配，保证各难度覆盖；
4. article_tags（文章-标签关联表）：文章表和标签表的链接表
5. questions（题目表）：对应文章的问题，注意字段**options遵循 JSON格式**，严格遵循`{"A":"选项内容","B":"选项内容","C":"选项内容"...}`，无多余引号/逗号 
6. question_attempts（答题记录表）：暂时不填，要求用户操作后自动记录，所有问题答完后自动计算分数，核心规则：`user_id+question_id`联合主键唯一，每个用户对每道题仅一条记录。
7. reading_history（阅读记录表）：暂时不填，要求用户浏览后自动创建
8. favorites（收藏表）：暂时不填，要求用户收藏后自动创建，核心规则：`user_id+article_id`联合主键唯一，每个用户对每篇文章仅一条收藏记录。
9. vocabulary_notes（单词本表） ：暂时不填，要求每创建一个用户自动创建一个单词，核心规则：每个用户的`word`字段不重复，贴合用户阅读的文章词汇难度。

##  三、转写

先在github中的database/sql文件中下载我的文件english_reading.sql，在sql文件中进行填写数据，归档原始文件，便于团队追溯。

然后要把已创建的数据库表结构 → `database/migrations/` 下的**迁移文件**（替代手动 SQL 建表）；

把待填写的测试数据 → `database/seeders/` 下的**填充文件**（替代手动 SQL 插入）；

**也就是最后这两个database/migrations/和`database/seeders/` 文件夹必须完成。**
